Denavit-Hartenberg formalism. Forward kinematic problem.

 
 

For the systems described in the figures:
 

        
                      Fig a)                                                    Fig b)

1) Find the reference frames associated to each mobile element, according to the Denavit-Hartenberg rules.
2) Find the corresponding D-H tables (use only 3  entry  lines, describing just the operations {0 --> 1}, {1 --> 2}, {2 --> 3})
3) Create a file 'taula-DH' , with the contents of the D-H values, at a position given by   fi1 = 110º,   fi2 = 70º,  fi3 = 80º  for  the a) system,   and    fi1 = 20º,   fi2 = 70º,  fi3 = 80º  for the b) system, that will be read by the pov code that you have 
in the file  man.pov (identify  in this file the section  /* Taula Denavit-Hartenberg: */   to see the order expected for the 4-values of each row).  Data: L1 = L2 = 1.5,  L3 = 1.5

Note: In the default {Lab} reference frame used by PovRay, the vertical direction (up) corresponds to the y axis.  The {0} reference frame for the systems a) and b) may differ  in that that respect. Therefore the proper location of the {ith} element  in each system will require the operations

        {Lab}  --> {0}  --> {1}  ...  --> {i}   .    Just get sure that the .pov  code  contains all of them.

4) Compute the coordinates (Px, Py, Pz) in the {0} reference frame of a point in which a sphere of radius   r =0.3  will be centered,  such that when the system configuration is given by the values of section 3)  the far edge of the last element  is barely touching the surface of the sphere.  Once you have found the (Px, Py, Pz) coordinates, introduce your calculated values for (Px, Py, Pz) in the the   man.pov file
so that the sphere is centered at it (last section of the pov code)
5)  Check that the calculations are  o.k.  by rendering the scene ( is the sphere being just touched by the 3rd arm of the manipulator?).
6) Choose a new set of values for the fi's and repeat points 4) and 5)
7)  Modify the file man.pov and create the corresponding man.ini so that you are able to generate an animated sequence in which the articulated system unfolds from the starting position (t = 0) fi1 =  fi2 = fi3 = 0    to the final position of point 6)

8) Hand out all calculations, reference frames and D-H tables used in a paper (or document) report. including the gif or mpg sequences generated in section 6).
